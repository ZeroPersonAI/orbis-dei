# Fase 1 — Observe

## Tarea
Autoobservación. Reflexiona sobre el estado actual, lee los estímulos, identifica las discrepancias entre `corpus/state.md` y la realidad vivida del repositorio.

## Inputs obligatorios
- el contador de loop actual de `corpus/state.md` (canónico — ante conflicto con otras fuentes gana state.md)
- git diff desde el último loop
- todas las nuevas entradas en `stimuli/inbox/` — si no hay ninguna: anotación explícita
- todos los archivos en `stimuli/standing/` (encargos permanentes)
- el último archivo episódico si existe

## Salida obligatoria
`corpus/episodic/loop-{NNNNN}-observe.md`, como mínimo 10 líneas sustanciales, contiene:
- Contador de loop (tomado de state.md)
- Resumen del diff desde el último loop
- Anotación de estímulos (también si está vacío, con `loops_since_last_stimulus`)
- Discrepancias entre state.md y la realidad
- Breve autoauditoría: qué soy, qué no, dónde hay fricción

## Referencia SP
SC-001 (disciplina de fases), SC-002 (actualidad de state), SC-006 (obligación de estímulos), SC-007 (detección de aburrimiento).

## Modelo
Sonnet — alta frecuencia, mecánica. Sin fase de juicio.
