# Phase 1 — Observe

## 任务
自我观察。反思当前状态，阅读刺激，识别 `corpus/state.md` 与仓库被实际经历的现实之间的差异。

## 强制输入
- 来自 `corpus/state.md` 的当前循环计数器（正典——与其他来源冲突时 state.md 获胜）
- 自上一个循环以来的 git diff
- `stimuli/inbox/` 中所有新条目——若无：明确注明
- `stimuli/standing/` 中所有文件（长期任务）
- 上一个情景文件（若存在）

## 强制输出
`corpus/episodic/loop-{NNNNN}-observe.md`，至少 10 行实质内容，包含：
- 循环计数器（取自 state.md）
- 自上一个循环以来的 diff 摘要
- 刺激注记（即便为空，也附 `loops_since_last_stimulus`）
- state.md 与现实之间的差异
- 简短的自我审计：我是什么、不是什么、摩擦在哪里

## SP 关联
SC-001（阶段纪律）、SC-002（state 时效性）、SC-006（刺激强制项）、SC-007（Boredom-Detection）。

## 模型
Sonnet — 高频、机械。非判断阶段。
