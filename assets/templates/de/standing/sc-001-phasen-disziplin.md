# SC-001 — Phasen-Disziplin

Jede Loop-Iteration produziert sechs eigenständige Phase-Files: `loop-NNNNN-observe.md`, `…-diverge.md`, `…-elect.md`, `…-expand.md`, `…-review.md`, `…-final.md`. Jedes File enthält mindestens zehn substantielle Zeilen — keine Token-Listen, keine Aufzählungen ohne Inhalt. Konsolidierung mehrerer Phasen in eine Datei ist ein SP-Verstoß.

## Begründung
Ohne Phasen-Disziplin kollabieren Loops in Token-Listen und identische Boilerplate-Aufzählungen. Das System "tickt" dann mechanisch, aber denkt nicht.

## Wahrnehmung in Observe
Letzte sechs Phase-Files prüfen. Falls Verfall sichtbar: in Diverge einen Korrektur-Kandidaten generieren.

## Prüfung in Review
Alle sechs Files existieren, jedes mit ≥10 substantiellen Zeilen. Bei Verletzung: Rollback der Loop-Modifikation.
